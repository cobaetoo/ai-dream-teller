# Next.js API Routes 및 DB 스키마 설계

- 너는 뛰어난 수익형 웹서비스 백엔드 기획자야
서버는 Next.js API Routes를 사용하고, DB Supabase를 사용할거야
우선 PRD 문서에 있는 내용과 context7 Next.js MCP를 기반으로 이 프로젝트의 서버 설계에 필요한 API 구조를 정리해서 PRD 문서를 업데이트 해
설계 시에는 MECE하게 기획하고, route 이름은 직관적으로 지어줘

- 이번에는 PRD 문서에 있는 내용과 Supabase MCP를 토대로 DB 스키마 등 구조를 설계해서 PRD 문서를 업데이트 해줘
마찬가지로 MECE하게 기획하고, 테이블 이름 및 칼럼 이름은 직관적으로 지어줘
DB Schema의 각 칼럼에 대해 Null 여부를 함께 표기해줘

- 현재까지 변경된 사항을 파악해서 repo에 commit push 해줘
commit 메시지는 변경된 사항을 요약해서 적어줘

- 현재 이 대화 세션에서 이루어진 모든 대화 파일을 있는 그대로 다운로드 받고 싶어. 파일명으로는 이 대화의 제목을 달아줘.
  해당 파일에서 AI의 응답은 삭제하고 유저가 입력한 프롬프트만 남겨줘. 해당 파일 내용에서 날짜와 시간으로 구분하는 내역은 모두
  삭제하고 본 문서에 대해 정리 요약한 내용도 필요 없이 오직 유저의 프롬프트만 리스트로 정리해. 마크다운 저장 경로는 현재
  프로젝트 내부의 `conversation_history` 안에 생성해.
